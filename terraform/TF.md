# Terraform

## Terraform best practices 

- Divide the configuration into multiple files.
- Utilize terraform fmt to verify the correct formatting of the files.
- Ensure the validity of the Terraform code by using terraform validate.
- Employ variables.tf to enhance token security.
- Compositions represent the final infrastructure composed of various Terraform modules. They should be fully configured within the composition itself. Avoid configuring the modules themselves using terraform.tfvars, as their variables should be configured at the top level by the end user of the modules.
- The .terraform directory is generated by the terraform init command, executed individually by each user who intends to utilize the Terraform configuration. Consequently, it is unique to each user.
- Treat the .tfstate file as sensitive data that should not be publicly shared or distributed among developers. It is more secure to use a "remote" backend to store it on a server, allowing multiple developers to collaborate on the same state.
- Limit the use of the terraform.tfvars file to compositions only.
